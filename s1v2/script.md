## [P2] 今日のお題 — 回転寿司大手3社のIR情報を比較してHTMLに

今日のお題はこちらです。営業やマーケティング、経営者にとっても重要なリサーチと資料作成。

**「回転寿司大手3社の最新のIR情報を比較分析して、見やすいHTMLにまとめてもらう」**

これを普通にやろうとすると、各社の決算資料を開いて、数字を拾って、表にまとめて、見栄えを整えて……半日はかかる作業ですよね。

こういう「調べものして、整理して、見せる形にする」という仕事こそ、AIに一番任せやすい領域です。

## [P3] プロンプト全文

実際に投げるプロンプトがこちらです。

> 回転寿司大手3社（スシロー・くら寿司・かっぱ寿司）の最新のIR情報をウェブでリサーチして、売上・営業利益・店舗数・客単価を比較分析してください。調査が終わったら frontend-design スキルを使って、経営者がパッと見て差がわかる白基調の見やすいHTML1枚にまとめてください。比較チャート付き、そのままプレゼンに使えるクオリティで。

3社を明示して、比較軸を4つ指定して、最後に `frontend-design` というスキルを呼び出してHTMLに整える、という内容です。

細かく指示を書く必要はありません。やってほしいことを、素直に日本語で書くだけで大丈夫です。

では、そのまま投げてみます。

## 【実演パート】Claude Codeデスクトップ版 で実行

（画面を Claude Codeデスクトップ版 に切り替え、プロンプトをペーストして送信）

送信すると、Claude Code がウェブリサーチを開始します。各社の公式IRページや決算説明資料を、順番に巡回していく様子が画面に流れていきますね。

……ここで、こういう**許可ダイアログ**が出てくることがあります。「このサイトを開いてもいいですか」「このファイルを作ってもいいですか」と聞いてくるものです。

これは、Claude Code が勝手にウェブにアクセスしたり、勝手にファイルを書き換えたりしないように、毎回ユーザーに確認してくる安全装置です。信頼できる操作なら許可、怪しければ拒否。

ちなみにこの調べもの中、私たちは他の仕事をしていて構いません。

……リサーチが終わると、次は `frontend-design` スキルが起動して、集めた数字をHTMLに組み上げていきます。このスキルについてはのちほど詳しくお伝えします。

（完成したHTMLをブラウザで開いて表示）

できました。売上、営業利益、店舗数、客単価が3社分並んで、比較チャートも入っています。白基調で、そのまま社内共有やプレゼンに出せる仕上がりですね。

プロンプトを書いてから、ここまでたった数分。私がやったのは、プロンプトを1回送っただけです。

完成した資料を見て、修正したい場合は、同様にClaude Codeに指示をするだけです。
