## [P2] 今日のお題 — 回転寿司大手3社のIR情報を比較してHTMLに

今日のお題はこちらです。

**「回転寿司大手3社の最新のIR情報を比較分析して、見やすいHTMLにまとめてもらう」**

これ、普通にやろうとすると、各社の決算資料を開いて、数字を拾って、表にして、見栄えを整えて……で、半日くらいかかる作業ですよね。

AIはウェブでの情報収集と、集めた情報の整理・レイアウト、この両方がとても得意です。今回はこの2つを、一気にやってもらいます。

## [P3] プロンプト全文

実際に投げるプロンプトがこちらです。

3社を明示して、比較軸を4つ指定して、最後に `frontend-design` というスキルを呼び出してHTMLに整える、という内容です。

細かく指示を書く必要はありません。やってほしいことを素直に言葉にするだけです。

では、そのまま投げてみます。

## 【実演パート】Claude.ai で実行

（画面を Claude.ai に切り替え、プロンプトをペーストして送信）

送信すると、Claude Code がウェブリサーチを開始します。各社の公式IRページや決算説明資料を順番に巡回していく様子が、画面に出ていきますね。

……ここで、こういう**許可ダイアログ**が出てくることがあります。「このサイトを開いてもいいですか？」「このファイルを作ってもいいですか？」と聞かれるやつです。

これは、Claude Code が勝手にウェブにアクセスしたり、勝手にファイルを書き換えたりしないように、毎回ユーザーに確認してくる仕組みです。信頼できる操作なら許可、怪しければ拒否。この感覚は、このあと触る CLI 版でも同じです。

……リサーチが終わると、次は `frontend-design` スキルが起動して、集めた数字をHTMLに組み上げていきます。

（完成したHTMLをブラウザで開いて表示）

できました。売上、営業利益、店舗数、客単価が3社分並んで、比較チャートも入っています。白基調で、そのままプレゼンに出せる仕上がりですね。

プロンプトを書いてから、ここまでだいたい10分。私がやったのは、プロンプトを1回送っただけです。

## [P4] クロージング — リサーチも資料作成もAIにまかせよう

営業、マーケティング、どんな職種の方も、リサーチと資料作成はAIにまかせて、どんどん生産性を上げていきましょう。

出力形式をHTMLにすると、自由度が一気に上がります。縦に長いLP風のページから、コンパクトなスライドまで、同じやり方で作れます。

実は、この講座のスライドも、すべてHTMLで作っています。
