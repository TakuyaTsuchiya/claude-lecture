## [P2] CLAUDEとは — AIが毎回読む「取扱説明書」

CLAUDE.mdとは、プロジェクトのフォルダに置いておく、ただのテキストファイルです。

特別なのは、Claude Code がセッション開始時に、このファイルを自動で読み込む、という点です。

たとえば、「一人称は『私』を使う」「文体はですます調で」、そういった指示を一度書いておくだけで、毎回同じ説明をしなくてよくなります。

設定しておけば、あとは自動で動いてくれます。

---

## [P3] まず3行でOK — 入門テンプレート

何を書けばいいかわからない、という方は多いですが、最初は3行だけでも十分です。

1行目はペルソナ。「あなたは○○会社のアシスタントです」のように、AIに役割を伝えます。

2行目は出力言語。「返答は必ず日本語で」と書くだけで、日本語が安定して返ってきます。

3行目はファイルの場所。よく参照する資料のパスを伝えておくと、毎回指定しなくてよくなります。

この3行があるだけで、Claude Code との対話がぐっとスムーズになります。

---

## [P4] ショートカット — `/init` で自動作成

手動で作ってもいいのですが、Claude Code には、CLAUDE.md を自動で作ってくれる便利なコマンドが用意されています。それが `/init` です。

`init` は `initialize` の略で、日本語にすると「初期化する」という意味です。もう少し噛み砕いて言うと、「最初のセットアップをしておいてね」とお願いするコマンドです。

このコマンドを実行すると、Claude Code がフォルダの中身を見て、そのプロジェクトにあった CLAUDE.md を自動で作ってくれます。

## [P6] 活用法 — ドキュメントのリンク集として使う

もう一つ、実務で一番使える使い方を紹介します。

ドキュメントのリンク集、つまりインデックスとして使う方法です。

社内に資料が複数ある場合、CLAUDE.mdに「会社概要はここ、商品カタログはここ」とリンクを書いておくだけで、Claude Code が必要なときに自動で参照しにいきます。

ファイルを毎回「これを読んでください」と渡す手間が、ゼロになります。

---

## [P7] クロージング — 書きすぎない

最後に、一つ注意点をお伝えします。

CLAUDE.mdは、書けば書くほど良くなるわけではありません。

AIは長いテキストを読むとき、後半ほど注意力が分散する傾向があります。重要な指示が大量のテキストに埋もれると、正確に守られなくなります。

目安は200行以内。増えてきたら、別ファイルに分けてCLAUDE.mdから呼び出すのが効果的です。

シンプルに保つほど、AIは的確に動いてくれます。
