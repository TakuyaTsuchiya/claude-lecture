## [P2] CLAUDEとは — AIが毎回読む「取扱説明書」

CLAUDE.mdとは、プロジェクトのフォルダに置いておく、ただのテキストファイルです。

特別なのは、Claude Code が会話を始めるたびに、このファイルを自動で読み込む、という点です。

つまり、「あなたは○○のアシスタントです」「返答は日本語で」、そういった指示を一度書いておくだけで、毎回同じ説明をしなくてよくなります。

家電製品の取扱説明書と同じイメージです。設定しておけば、あとは自動で動いてくれる。

---

## [P3] まず3行でOK — 入門テンプレート

何を書けばいいかわからない、という方は多いです。最初は3行で十分です。

1行目はペルソナ。「あなたは○○会社のアシスタントです」のように、AIに役割を伝えます。

2行目は出力言語。「返答は必ず日本語で」と書くだけで、日本語が安定して返ってきます。

3行目はファイルの場所。よく参照する資料のパスを伝えておくと、毎回指定しなくてよくなります。

この3行があるだけで、Claude Code との対話がぐっとスムーズになります。

## [P5] 活用法 — ドキュメントのリンク集として使う

もう一つ、特に非エンジニアの方に実感していただきやすい使い方を紹介します。

ドキュメントのリンク集、つまりインデックスとして使う方法です。

社内に資料が複数ある場合、CLAUDE.mdに「会社概要はここ、商品カタログはここ」とリンクを書いておくだけで、Claude Code が必要なときに自動で参照しにいきます。

ファイルを毎回「これを読んでください」と渡す手間が、ゼロになります。

---

## [P6] クロージング — 書きすぎない

最後に、一つ注意点をお伝えします。

CLAUDE.mdは、書けば書くほど良くなるわけではありません。

AIは長いテキストを読むとき、後半ほど注意力が分散する傾向があります。重要な指示が大量のテキストに埋もれると、正確に守られなくなります。

目安は10行以内。増えてきたら、別ファイルに分けてCLAUDE.mdからリンクを張るのが効果的です。

シンプルに保つほど、AIは的確に動いてくれます。
