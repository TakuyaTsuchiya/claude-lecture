## [S1] P1 カバー — 【畑さん向け】LMSアップロードSkillレクチャー

### ト書き
- **開始状態**: スライド P1 が画面いっぱいに表示されている
- **アクション**: 動かさず、ナレーションのみ

### ナレーション
畑さん向けのミニレクチャーです。

WisdomBase LMSへのコース構築、毎回手作業でやっていると、動画レクチャーの登録、確認クイズの登録、トラック制限の設定で結構な時間が取られますよね。

これを、Claude Codeから自動化できるSkillを作りました。その使い方を3分でご紹介します。

### 終了状態

## [S2] P2 STEP 1 — インストール手順

### ト書き
- **開始状態**: スライド P1 が表示中
- **A. スライドめくり**: 右矢印キー（または Space）で **P2 に進める**
- **B. 概要ナレーション**: スライドを見せたまま、2コマンドの説明

### ナレーション（B区間）
まずはインストール。Claude Code の中で2コマンド打つだけです。

1つ目が `plugin marketplace add`。私のGitHubリポジトリをマーケットプレイスとして登録するコマンド。

2つ目が `plugin install`。そこからプラグイン本体を入れるコマンドです。

では実際にやってみます。

### ト書き続き
- **C. 画面切り替え**: Cmd + Tab（または Mission Control）で **Claude Code デスクトップ版 にフォーカス**
- **D. 1コマンド目を実行**:
  - プロンプト入力欄をクリック
  - `/plugin marketplace add TakuyaTsuchiya/wisdombase-lms-skill` を**タイプ**（Screen Studio に打鍵が映る）
  - Enter
  - 承認プロンプトが出たら **Yes（または「はい」）を選択**
- **E. 2コマンド目を実行**:
  - 続けて `/plugin install wisdombase@wisdombase-lms-skill` を**タイプ**
  - Enter
  - 承認プロンプトが出たら **Yes を選択**
  - 「installed」「completed」など完了表示が出るまで待つ

### ナレーション（D-E区間）
1つ目のコマンドを入力。承認はYes。

2つ目のコマンド。これも承認はYes。

完了です。これで `/wisdombase` というスラッシュコマンドが使えるようになります。

### ト書き続き
- **F. 画面切り替え**: Cmd + Tab で **スライド（Chrome）に戻す**
- **G. スライド P2 のまま**で次のシーンへ

### 終了状態
- スライド P2 が表示中

## [S3] P3 STEP 2 — 実行手順

### ト書き
- **A. スライドめくり**: 右矢印キーで **P3 に進める**
- **B. ナレーション**: スライドの①〜④を **指差すように** 順に話す（Screen Studio のカーソルが補助になる）

### ナレーション
使い方の流れを押さえておきます。

**①番**、Claude Code に「WisdomBase にマンガ生成講座のモジュール2をアップしたい」みたいに、自然言語で話しかけてください。もちろん `/wisdombase` とスラッシュコマンドで打っても起動します。

**②番**、起動すると Claude から順番に質問が飛んできます。コースID、動画タイトル、クイズの内容、こうしたデータを答えていくだけです。

**③番**、Claude が必要な情報を集め終わると、スクリプトファイルが自動で生成されます。保存先は環境によって変わるので、Claude が画面に表示してくれるパスを確認してください。

**④番**、最後に、そのスクリプトを WisdomBase の管理画面で実行する。これで完了です。

ポイントは、Claude がゼロからスクリプトを書くわけではない、ということ。こちらで用意したテンプレートに沿って書くようになっているので、毎回同じ品質のスクリプトが出ます。過去にハマった罠も全部回避済みです。

### 終了状態
- スライド P3 が表示中

## [S4] P4 参考 — 生成されるスクリプトの例

### ト書き
- **A. スライドめくり**: 右矢印キーで **P4 に進める**
- **B. ナレーション中の動作**: ★マーク（オレンジの色付き部分）を **マウスカーソルで指す** とわかりやすい
  - `COURSE_ID = 47002` を指す
  - `LECTURES = [...]` を指す

### ナレーション
実際に生成されるスクリプトはこんな感じです。

たとえば「マンガ生成講座のモジュール2、動画3本」と依頼すると、コースIDが `47002`、レクチャータイトルが3つ並んだスクリプトが出来上がります。

★ で囲まれた部分、ここがClaudeが依頼内容に応じて埋めるパラメータです。コースIDと、動画タイトル一覧。それ以外の処理ロジックはテンプレ固定なので、毎回同じものが出ます。

### 終了状態
- スライド P4 が表示中

## [S5] P5 参考 — DevToolsでの実行画面

### ト書き
- **A. スライドめくり**: 右矢印キーで **P5 に進める**
- **B. ナレーション中の動作**:
  - 「F12」「Cmd + Option + I」と言ったときに該当箇所を **指す**
  - 「Console タブ」と言ったときに スライド画像内の **Console タブを指す**
  - 「オレンジの枠」「`>` プロンプト」のところで、スライド画像の **オレンジ枠を指す**

### ナレーション
最後に、生成されたスクリプトの貼り付け先です。

WisdomBase のコース編集画面を開いた状態で、F12キー、Macなら `Cmd + Option + I` で DevTools を開きます。

右側の Console タブを選択。

このオレンジの枠で囲ったあたり、「SES Removing...」というメッセージの下にある `>` プロンプト、ここにスクリプトを貼り付けて Enter を押すだけです。

実行すると、コンソールにログが流れて、レクチャーIDの一覧が表示されます。それを次のステップ、たとえば動画アップロードや、クイズ作成のスクリプトの入力として使う、という流れになっています。

### 終了状態
- スライド P5 が表示中

## 撮り直しのときに気をつけること

- **スライドの位置を間違えない**: P1 から P5 まで順に進む。戻るときは左矢印
- **Claude Code のセッションを毎回クリアする**: 「インストール済み」状態だと演出が崩れる。事前に `/plugin uninstall wisdombase@wisdombase-lms-skill` で消しておく
- **Screen Studio の自動ズーム**: 重要な操作（コマンド入力、Yes選択）にズームが乗るよう、操作前に **数百ミリ秒間を取る**
- **ナレーションの間**: 区切りごとに **0.5〜1秒の間** を入れる。後から字幕を入れやすくなる
- **失敗時の復旧**: もし2コマンド目で承認をNoしてしまったら、もう一度 `/plugin install` を打ち直す。動画はその場面だけカットすればよい
