## [P1] カバー — 確定申告Skillで学ぶSkillsの極意

S3では、Claude Codeを業務で使いこなすための2つの武器、SkillsとMCPをご紹介します。

まずは「Skills」です。インストールするだけで、その道のプロの手順がそのまま動く——その仕組みを、確定申告の自動化Skillを題材に体験していただきます。

---

## [P2] Skillsとは — 専門家の手順をパッケージ化したもの

まず、Skillsがない世界を想像してください。

確定申告。毎年3月が近づくと憂鬱になる、あの作業です。領収書の整理、帳簿の入力、控除の計算、申告書の作成——正確にやろうとすると、税理士に依頼することになり、費用は年間数万円から十数万円かかります。

Skillsは、この問題を解決する仕組みです。

一言で言うと、**専門家の手順をパッケージ化したもの**。インストールするだけで、専門家の判断プロセスがそのまま使える状態になります。

今日使うのは「shinkoku」。確定申告の専門家の手順が丸ごとパッケージになったSkillです。スライドに並んでいる通り、申告要否の判定から始まり、領収書の仕訳登録、決算整理、税額計算、そして最後は国税庁のサイトへの自動入力まで——一本のパイプラインで動きます。

作った人の知識が、インストールした全員に届く。これがSkillの本質です。

## 【実演パート】確定申告Skillを動かす

（スライドを閉じて、ターミナル画面に切り替え）

Skillのプラグイン管理は、現時点では**Claude Code CLI専用**です。デスクトップ版のチャットUIではインストールコマンドが動きません。ターミナルで進めます。

（Claude Code CLIを起動）

インストールは3ステップです。まずshinkokuのマーケットプレイスを追加します。

> /plugin marketplace add kazukinagata/shinkoku

続いてインストールします。

> /plugin install shinkoku@shinkoku

スコープは「Install for you」——自分のユーザー環境で使えるものを選びます。

> /reload-plugins

1点だけ強調します。Skillは**新しいセッションを開始するタイミングで読み込まれる**仕様です。インストール直後の同じセッションでは使えません。必ず一度セッションを終了して、新しいセッションを開き直してください。

（新しいセッションを開く）

まず `/setup` で初期設定を行います。マイナンバーや住所など個人情報を入力するステップです。実際に使うときは、人に画面を見せない環境で事前に済ませておくのがおすすめです。

（/setup を実行）

対象年度、事業所得の有無、申告の種類——こんな形で質問に答えていきます。まだ続きますが、一度設定しておけば次からは設定ファイルを引き継いで動きます。

（質問をいくつか見せた後、設定完了の画面へ）

設定完了です。`shinkoku.config.yaml` と `shinkoku.db` が生成されました。YAMLは設定を書くテキストファイル、DBはデータベースファイルです。仕訳や進捗データはここに蓄積されていきます。

では、今日の見せ場に入ります。

（/assess を実行）

> /assess

assessは英語で「評価する・査定する」という意味です。確定申告が必要かどうかを評価・判定するコマンドです。

（質問に答えていく）

給与所得はあるか、事業収入はおよそいくらか——10問ほど質問に答えると……

（判定結果が出る）

判定が出ました。「申告が必要です」。さらに、住民税の申告についても補足が出ています。自分の状況に合わせた判定を、プロンプト1行ではなく、専門家の判断フローに沿って出してくれます。

この後のパイプラインも紹介します。`/journal` でレシートや領収書の画像を渡すと、勘定科目を自動判定して帳簿に登録されます。`/settlement` で決算整理、`/income-tax` で税額計算と還付額の確認。そして最後の `/e-tax` で、Claudeがブラウザを操作して国税庁の確定申告書作成コーナーに自動入力します。マイナンバーなど個人情報のみ手動で入力すれば、申告が完了します。

税理士に依頼していた一連の作業が、インストールするだけで動く。これがSkillの力です。
