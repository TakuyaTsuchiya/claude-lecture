## 【実演パート①】ステータスラインを確認する

まず最初に、トークン消費を管理するステータスラインを確認してみましょう。

（画面を Claude Code デスクトップ版に切り替え）

まずはこの丸いアイコンをクリックしてみてください。

（ステータスラインを指し示す）

これが「ステータスライン」です。表示される項目を順番に見ていきましょう。

まず左から、**現在のコンテキストウィンドウの使用率**。今このセッションで作業台の何パーセントを使っているかが見えます。

次に、**5時間の制限**。直近5時間で使えるトークン枠のうち、今何パーセント消費したかを示します。100%に達すると、枠がリセットされるまでモデルが制限されます。また、あと何時間でリセットされるかも表示されているので、「あとN時間待てば回復する」とひと目でわかります。

その隣が**週次・全モデルの制限**。7日間の総消費量です。

そして**週間・Claude Design の制限**。Claude Design 機能を使った週次の消費量です。

そして**Sonnetのみ**。SonnetとOpusは用途が異なるため、Sonnetの使用量は独立した枠で管理されています。

（各項目を指し示しながら説明する）

ここまでは「何かを消費している」という体感の話でした。その「何か」の正体を、次に見ていきましょう。

## 【実演パート②】OpenAI Tokenizerでトークンを体験する

トークンとは、AIがテキストを処理するときの最小単位です。実際に目で見てみましょう。

（ブラウザで OpenAI Tokenizer を開く）

OpenAI が公開している Tokenizer というツールを使います。

テキストエリアに日本語を入力してみます。「今日の会議の議事録をまとめてください」と入力しましょう。

（テキストを入力する）

色分けされましたね。この1つひとつの色の塊が「トークン」です。日本語だと、1文字ないし数文字ごとにひとつのかけらになっていることがわかります。

今度は英語で試してみます。「Please summarize the minutes.」と入力してみると…こちらは単語ごとに区切られていて、かけらの数がずっと少ないですね。日本語は英語よりずっと多くのトークンを消費します。

（英語テキストを入力して比較する）

先ほどステータスラインで見た数字は、こうした細かいかけらの積み上げの総量です。Claude Code に何かを伝えるたびに、この数字は増えていきます。

## [P2] コンテキストウィンドウとは — AIの「作業台」は有限

もう一つの概念、「コンテキストウィンドウ」を説明します。

コンテキストウィンドウとは、AIが一度に見ることができる範囲のことです。作業台に例えるとわかりやすいです。

あなたが Claude Code と会話するたびに、その履歴が作業台の上に積み上がっていきます。読み込んだファイルも、AIが生成したコードも、すべてが台の上に乗ります。

作業台には上限があります。満杯に近づくと、AIは古い情報を参照できなくなり、「最初の指示を忘れたような振る舞い」が起き始めます。応答の品質が静かに落ちていくんです。

## [P3] 実践ルール — 作業が終わったら New Session

では、どうすればいいか。答えはシンプルです。

1つのタスクが終わったら、すぐに新しいセッションを始める。これだけです。

（画面の New session ボタンを指し示す）

左上の「New session」を押すと、作業台がリセットされます。きれいな状態で始めれば、AIは次のタスクに集中できます。逆に、終わったタスクの会話履歴を引きずったまま別の仕事を頼むと、注意が分散して精度が下がります。

1セッション1タスク。これを習慣にしてください。

## [P4] クロージング — コンテキストとトークンのマネジメント

今回お伝えしたかったことを一言でまとめます。

Claude Code で業務するということは、コンテキストウィンドウとトークンをマネジメントすることだ、ということです。

ステータスラインでトークン消費を確認し、タスクが終わったらすぐに New Session。この2つの習慣を身につけるだけで、Claude Code をより使いこなせるようになります。
