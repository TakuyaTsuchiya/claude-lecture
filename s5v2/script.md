# S5V2 台本 — Vercel入門 アカウント作成と初めてのデプロイ

- **想定尺**: 約5分（うち実演 約3.5分）
- **実演**: あり
- **成果物**: Vercel上に公開された `my-first-repo` のWebページ

---

## [P1] カバー — Vercel入門 アカウント作成と初めてのデプロイ

前回、GitHubにコードの保管庫を作りました。でも、保管庫に置いただけでは、まだ誰もWeb上で見ることはできません。コードをWebサイトの形で世界に公開するには、もう1つ道具が必要です。それがVercelです。

---

## [P2] VercelはWebサイトを公開する窓口

Vercelは、GitHubに置いたコードを、Webサイトとして世界に公開してくれるサービスです。noteに記事を書くと自動で公開URLが発行されますよね。あれと同じ感覚で、GitHubのリポジトリをVercelに渡すと、世界からアクセスできるURLを作ってくれます。

今日やることは3つです。Vercelのアカウントを作り、GitHubと連携し、前回作った `my-first-repo` を選んでデプロイします。デプロイというのは、簡単に言えば「公開ボタンを押す」ことです。

---

## [P3] GitHub と Vercel

GitHubとVercelは役割が違います。GitHubはコードを置いておく保管庫です。一方、Vercelはその保管庫の中身を読み取って、Webサイトとして世界に出してくれる公開サーバーです。2つを1回つないでしまえば、あとは `git push` でコードを更新するだけで、Vercelが自動で最新のサイトに差し替えてくれます。

自分で公開サーバーを用意する必要も、更新のたびにアップロードし直す必要もありません。ここがVercelの気持ちよさです。

---

## [P4] プロンプト全文

まずは、公開するためのWebページを用意します。前回作った `my-first-repo` には `README.md` しか入っていないので、このままVercelにデプロイしても表示できる画面がありません。Claude Codeに、シンプルなindex.htmlを1枚作ってもらいます。

プロンプトはこちらです。長いので概要欄にも貼っておきます。

> my-first-repo に index.html を追加してください。「Claude Codeと作るはじめてのWebページです」という見出しと、短い挨拶文を入れた1画面のページにしてください。完了したらコミットとプッシュまでお願いします。

---

## 【実演パート】index.html追加 → Vercelデプロイ → 公開URL確認

（Claude Code デスクトップ版に切り替え、プロンプトを送信）

Claude Codeに先ほどのプロンプトを送ります。index.htmlを作り、コミットし、プッシュまで自動で進みます。

（ブラウザで https://vercel.com を開く）

次にVercelです。公式サイトにアクセスして「Sign Up」をクリックします。GitHubアカウントでそのままサインアップできるので「Continue with GitHub」を選びます。権限の確認画面が出たら Authorize を押してください。

（Vercelのダッシュボード画面）

ダッシュボードに入ったら、新しいプロジェクトを追加します。先ほどGitHubと連携したので、自分のリポジトリ一覧がそのまま表示されます。`my-first-repo` を選んでImportをクリックしてください。

（プロジェクト設定画面）

設定画面が出ますが、ここは触らなくて大丈夫です。そのまま Deploy ボタンを押します。30秒ほどでビルドが終わり、公開URLが発行されます。

（発行されたURLをクリック、ブラウザで開く）

URLをクリックすると、先ほどClaude Codeに作ってもらったページが、世界中からアクセスできる状態で表示されます。これで、自分のWebページが世界に公開できました。
