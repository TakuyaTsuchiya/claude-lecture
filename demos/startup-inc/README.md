## 1週間のストーリーライン（2026-04-06（月）〜2026-04-10（金））

4層構造HTMLで「顧客対応動向→社内進捗→経営方針→示唆」の連鎖が見える形で仕込む。

- **顧客対応動向**：#cs-support・#sales-voice で「決済機能がほしい」の声が週内5件集まる
- **社内進捗**：#product-dev で決済API連携の技術調査を開始、#sales で見込み案件の決済機能ニーズがクロージングの障壁になっている
- **経営方針**：Notion経営会議議事録DBで「Q1新機能として決済機能を承認」
- **示唆**：次週の焦点は「Q1ロードマップ詳細化・顧客への告知タイミング設計」

## 4層構造HTMLの狙い

受講者がPoC動画を見た瞬間に「顧客の声が経営判断に連鎖する1枚絵」が立ち上がるように、4層を上から下へ読むと経営判断の流れが見えるよう設計する。

```
顧客対応動向   ← Slack #cs-support・#sales-voiceから抽出
  ↓
社内進捗       ← Slack #product-dev・#sales から抽出
  ↓
経営方針       ← Notion経営会議議事録DBから抽出
  ↓
示唆           ← 上記3層から導出（来週のフォーカス）
```
