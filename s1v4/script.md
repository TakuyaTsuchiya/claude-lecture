## [P2] 今日のお題 — アジェンダMarkdown1枚から会議用HTMLサイトを作る

用意したサンプルデータはこちらです。

架空のディスカウントストア「驚安堂」が開く、2026年夏の新商品リリース説明会のアジェンダです。PBブランド「マジ価格NEO」の10商品を一斉投入する夏商戦、その説明会のアジェンダをMarkdownで書いたものです。

このMarkdownファイル1枚を Claude Code に渡して、URLを送るだけで参加者全員がブラウザで開ける会議資料HTMLに変換する、それが今日のゴールです。

## [P3] プロンプト全文

実際に投げるプロンプトがこちらです。

> @agenda.md のアジェンダをもとに、社内会議用のHTMLページを1枚作ってください。URL1本で参加者が開けるよう、スタイルも込みの自己完結HTMLで。frontend-design スキルを使って見栄えよく仕上げてください。

`@agenda.md` というのは、ファイルを直接参照する書き方です。Claude Code でフォルダを開いた状態でこう書くと、そのフォルダにあるファイルをそのまま渡せます。

あとは何を作ってほしいか、日本語で書くだけ。細かい指定は不要です。

では実際にやってみます。

## 【実演パート】Claude Code で HTML を生成

（画面をClaude Codeデスクトップ版に切り替え、agenda.mdがあるフォルダを開く）

まず「新しいプロジェクト」からフォルダを選択します。agenda.md が入っているフォルダを開きます。

（プロンプトをペーストして送信）

送信すると、まず `frontend-design` スキルが自動で読み込まれます。このスキルが、見栄えのいいHTMLを出力するための設計ガイドラインをClaude Code に渡してくれます。

あとはコードを書いてくれるのを待つだけです。

（HTMLが生成されるまで待機）

できました。生成された `index.html` をブラウザで開いてみましょう。

（ブラウザで index.html を開いて表示）

ヒーローセクション、アジェンダ一覧、KPI、スケジュール……Markdownに書いた内容がすべてレイアウトされています。これをそのままURLで送れば、参加者全員がブラウザで開けます。

プロンプトを1回送っただけで、ここまで仕上がりました。

## [P4] クロージング — 設定より、まず手を動かそう

はじめてClaude Code を使ってみて、どうでしたか？

「`@` の使い方が分からなかった」「frontend-design って何だろう」「もっと細かく指示しないといけないのかな」そう感じた方もいるかもしれません。

全部、あとから覚えれば十分です。

Claude Code の使い方を覚えるのに一番いい方法は、使いながら覚えることです。分からないことが出てきたら、そのままClaude Code に「これはどういう意味ですか」と聞けばいい。詰まったら「こういうエラーが出たのですが」と貼り付けて聞けばいい。

設定を全部理解してから動こうとすると、それだけで止まってしまいます。

まず手を動かして、詰まったらその都度調べる。それが、Claude Code を一番速く使えるようになる道です。
