# S5V6 台本 — 顧客管理アプリのフロントエンドをスペック駆動で作ろう

- **想定尺**: 約5分（うち実演 約3.5分）
- **実演**: あり

---

## [P1] カバー — 顧客管理アプリのフロントエンドを作ろう

この動画では、顧客管理アプリ、いわゆるCRMと呼ばれる領域のアプリをスペック駆動で開発します。Claude Codeでアプリ開発用のCLAUDE.mdと、仕様書であるspec.mdを作り、それを元に実装まで行います。顧客の一覧、商談の管理、見込みから成約までのステータス遷移を、Claude Codeと協働で組み上げることがゴールです。

---

## [P2] TODAY'S GOAL — スペック駆動でアプリ開発を体験する

流れは3ステップです。アプリ開発用のCLAUDE.mdを書き、spec.mdを書いて画面と機能を決め、Claude Codeと協働でアプリを組み上げます。

アプリ開発にはスペック駆動という考え方があります。先に仕様書を書き切ってから実装に入る進め方です。Claude Codeと仕事をするときは、いきなりプロンプトから書き始めるのではなく、CLAUDE.mdとspec.mdで土台を用意するのが鍵です。Claude Codeは渡されたドキュメント通りに動くので、ドキュメントの質がそのまま出力の質になります。

---

## [P3] WHY — 月額数千円のSaaSが自前で作れる時代

顧客管理のSaaSに月額で払っている会社は多いです。ただしアプリの機能の核は、データを登録・一覧・編集・削除する4つの操作、CRUDです。そこに顧客と商談の紐付けや、ステータスの遷移が乗っているだけです。

手元で動くアプリを作ることで、こうしたSaaSがどのような仕組みなのかを学びます。この核だけなら、スペック駆動とClaude Codeの組み合わせで1時間で動くものが完成します。

---

## [P4] CLAUDE.md — アプリ開発用CLAUDE.mdに書くこと

（画面を Claude Code デスクトップ版に切り替え、新しいプロジェクト用ディレクトリを作成）

アプリ開発用のCLAUDE.mdに記載するのは技術スタック、ディレクトリ構成、コーディング規約、データ構造、デザイン規約、やってはいけないこと、の6項目。わからないことが多いと思いますが、とりあえず進めて動くものを作り、あとから振り返ってください。概要欄にCLAUDE.mdを貼っておきます。

（CLAUDE.mdファイルを画面に表示）

続いてClaude Codeとspec.mdを書きます。Claude Codeが迷わず実装できるよう、顧客と商談が1対多で紐付くデータ構造、見込み・提案・成約の3列で並ぶカンバン画面、ドラッグでステータスを動かす操作、localStorageに保存する方針を言語化し、仕様書を作成します。

---

## [P5] PROMPT

（スライドをP5に切り替え、プロンプトを読み上げ）

> CLAUDE.md と spec.md を読んで、
> 顧客管理アプリを実装してください。
>
> index.html / app.js / styles.css の
> 3ファイルで完結させてください。
>
> データはブラウザのlocalStorageに保存します。
> 実装が終わったらGitHubにプッシュしてください。

次に、実装を開始するプロンプトをClaude Codeに渡します。CLAUDE.mdとspec.mdに詳細が明記されているので、ここでは「実装して」と頼むだけで済みます。

（Claude Codeにプロンプトを投入。ファイルが生成されていく）

実装が終わったら、ブラウザで開いて動かしてみます。

（実装したアプリを開き、顧客と商談の登録、カード選択、メモ追記、カンバンでステータスをドラッグ移動、削除まで順に操作）

顧客と商談がリスト化され、見込みから成約までのカードが3列のカンバンに並びます。カードを右の列にドラッグすればステータスが進みます。データはブラウザのlocalStorageに入っているので、リロードしても消えません。

続いて、GitHubへのプッシュもClaude Codeに頼みます。

> 実装が終わりましたので、GitHubにpushしてください。

（Claude Codeがpushを実行）

GitHubへプッシュすると、Vercelで自動デプロイが走り、URLが発行されます。

（発行されたURLを開く）

## [P6] クロージング — SaaSのフロント、完成しました

フロントエンド部分のみ開発したので、このアプリのデータはあなたのブラウザの中にしか残らず、他の人と共有できません。S6でSupabaseのデータベースを使うと、どの端末からでも同じデータが見えるようになります。

これで、スペック駆動で顧客管理アプリを作ることができました。
