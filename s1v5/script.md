## [P1] カバー — Claude Code を安全に使う方法を知ろう

ここまで、Claude Codeの基本的な使い方を体験してきました。

今回はその続きとして、「権限モードの使い分け」と「入力情報のあつかい方」の2つをお伝えします。

どちらも最初に把握しておくと、安心して使い続けられるようになります。

---

## [P2] Claude の操作は2種類ある

Claude Code は主に「ファイルの編集」と「コマンド実行」を行います。

**ファイル編集**は、コードを書いたりテキストを変えたりする操作です。Claude が直接ファイルを読み書きするだけなので、影響範囲が明確です。

一方、**コマンド実行**は `npm install` や `git push` のように、ターミナルでコマンドを走らせる操作です。ネットワーク通信が起きたり、ファイルが削除されたりと、ファイル編集より影響範囲が広くなります。


---

## [P3] 3つの権限モード

Claude Code には、どこまで自動で進めてよいかを切り替える「権限モード」があります。

**「許可を確認」**は、ファイル編集もコマンド実行も毎回確認が入ります。使い始めの期間に適しています。

**「編集を承認」**は、ファイル編集は自動で進み、コマンド実行だけ確認が入ります。これがデフォルトで、日常作業ではこのモードをメインに使います。

**「プランモード」**は、ファイルの変更もコマンドの実行も行わず、計画だけを提示します。大きな作業を依頼するとき、先にどう進める予定かを確認してから承認する、という使い方をします。

---

## 【実演パート①】モードの切り替え

（画面を Claude Code デスクトップ版に切り替え）

モードの切り替えは、画面左下のアイコンから行います。

（アイコンをクリックしてモードメニューを開く）

「許可を確認」「編集を承認」「プランモード」の3つが並んでいます。

普段は「編集を承認」のまま使い、複雑な作業の前だけプランモードに切り替える、というのが基本的な使い方です。

---

## [P4] 確認が必要なときはこの画面が出る

「許可を確認」または「編集を承認」モードでコマンド実行が発生すると、この確認画面が表示されます。

**「拒否」**で操作を止める、**「一度だけ許可」**でその1回だけ実行する、**「常に許可」**で以降も自動で実行します。

「常に許可」を押すと次回から確認が省略されます。よく使うコマンドには便利ですが、内容を読んでから判断するようにしてください。

---

## [P5] 残り2つは普段触らなくていい

モード一覧には「許可をバイパス」と「自動モード」もありますが、どちらも初期状態では選べません。Claude Code の設定で有効化が必要です。

今は気にしなくて大丈夫です。

---

## [P6] 入力内容のあつかい

機密情報と個人情報の扱いについてお話しします。

プロンプトはAnthropicのサーバーを経由しますので、パスワード、APIキー、顧客の個人情報、社内外の機密文書といった情報は、Claude Codeに入力しないでください。


---

## 【実演パート②】プライバシー設定の確認

（ブラウザで claude.ai を開く）

デスクトップ版のアイコンから「設定」を開きます。

（Settings → プライバシーに移動）

「Claudeの改善にご協力ください」という項目があります。これがオンになっていると、会話がモデルの改善に利用される可能性があります。必要に応じてオフにしてください。

（設定画面を見せて閉じる）

---

この3つを意識するだけで、Claude Code を安全に使い続けられます。
