## [P1] カバー — モードを使い分けよう

V1からV4で、Claude Codeの基本的な使い方を体験してきました。

今回はその続きとして、2つのことをお伝えします。「権限モードの使い分け」と「入力情報のあつかい方」です。

どちらも最初に把握しておくと、安心して使い続けられるようになります。

---

## [P2] 3つのモード

Claude Code には、操作に対してどこまで確認を求めるかを切り替える「権限モード」があります。

実際の画面でも確認できますが、よく使うのは3つです。

**1つ目は「許可を確認」**。ファイルの編集もコマンドの実行も、毎回確認が入ります。Claude Code を使い始めたばかりの期間に適しています。

**2つ目は「編集を承認」**。これがデフォルトです。ファイルの読み書きは自動で進み、シェルコマンドの実行だけ確認が入ります。日常の作業ではこのモードをメインに使います。

**3つ目は「プランモード」**。ファイルの変更もコマンドの実行も一切せず、計画だけを提示します。「この依頼、本当にこの方針で進めていいか」を確認したいときに切り替えます。

---

## 【実演パート①】モードの切り替え

（画面を Claude Code デスクトップ版に切り替え）

モードの切り替えは、画面左下のアイコンから行います。

（アイコンをクリックしてモードメニューを開く）

「許可を確認」「編集を承認」「プランモード」の3つが並んでいます。番号キーの1・2・3でも直接切り替えできます。

（1→2→3と切り替えて見せ、2に戻す）

普段は「編集を承認」のまま使い、複雑な作業の前だけプランモードに切り替える、というのが基本的な使い方です。

下の2つ「許可をバイパス」と「自動モード」は、設定で有効化しないと選べません。通常の作業では使いません。

---

## [P4] 入力内容のあつかい

次に、入力内容のあつかいについてお話します。

**まず覚えておいてほしいのは、プロンプトはAnthropicのサーバーを経由するという点です。** パスワード、APIキー、顧客の個人情報、社内の機密文書といった情報は、Claude Codeに入力しないでください。

もう1点、学習への利用についてお話します。Claude Code デスクトップ版はデフォルトでモデルの学習に使われません。

## 【実演パート②】プライバシー設定の確認

（ブラウザで claude.ai を開く）

`claude.ai` にアクセスして、右上のアカウントメニューから「設定」を開きます。

（Settings → プライバシーに移動）

「Claudeの改善にご協力ください」という項目があります。これがオンになっていると、会話がモデルの改善に利用される可能性があります。必要に応じてオフにしてください。

（設定画面を見せて閉じる）

設定場所だけ把握しておけば十分です。

この3つを意識するだけで、Claude Code を安全に使い続けられます。
