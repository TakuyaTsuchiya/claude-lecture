## [P1] カバー — Claude Code を安全に使う方法を知ろう

V1からV4にわたって、Claude Code の基本操作を体験してきました。

ここで一度立ち止まって、安全に使うための基本をお伝えします。

Claude Code は強力なツールですが、使い方次第でファイルの消失や予期せぬ大量の操作につながることがあります。特に「なんとなくEnterを押す」という習慣が、思わぬ事故を引き起こします。

この動画を見終わったあとで、安心してS2以降の応用に進んでもらえる状態を作るのがゴールです。

## [P2] 権限3段階 — 権限モードを使い分けよう

Claude Code には、権限モードが3段階あります。

スマートフォンで「このアプリにカメラへのアクセスを許可しますか？」と聞かれる仕組みと同じ考え方です。Claude Code も、操作内容に応じてユーザーの許可を求めます。

**1つ目はデフォルトモード**。ファイルの編集も、コマンド実行も、毎回確認を求めます。慣れるまでの最初の期間は、このモードで使うことをおすすめします。

**2つ目はAuto-accept editsモード**。ファイルの読み書きは自動で進みますが、シェルコマンドの実行は引き続き確認を求めます。日常の業務でよく使う設定です。

**3つ目はフルオートモード**。すべての操作が確認なしで実行されます。後ほど説明しますが、このモードは使いどころを選ぶ必要があります。

## 【実演パート①】権限モードの切り替え

（画面を Claude Code デスクトップ版に切り替え）

Claude Code の右下にあるメニューから権限モードを切り替えられます。

今はデフォルトモードになっていることを確認します。実際に操作してみましょう。

（デフォルト → Auto-accept edits → フルオートと切り替えて見せ、デフォルトに戻す）

これだけで切り替えできます。作業の内容や場面に合わせて使い分けてください。

## 【実演パート②】承認画面の読み方とCtrl+E

次に、承認画面の読み方です。

（Claude Code に何か操作をさせて承認ダイアログを出す）

Claude Code が操作を提案すると、このような確認ダイアログが表示されます。

ここで「Enterだけ押して先に進む」という癖が一番危険です。利用規約を読まずに「同意する」を押し続けるのと同じことが起きます。

まず、**何をしようとしているのかを読む**。そして、迷ったら **Ctrl+E** を押してみてください。

（Ctrl+Eを押す）

Claude Code が「なぜこの操作を提案しているのか」を日本語で説明してくれます。この解説を読んで、意図を理解してから判断する癖をつけましょう。

---

## 【実演パート③】rm -rf系コマンドの挙動

最後に、破壊的コマンドの実演です。

（ダミーディレクトリを事前に用意しておく）

ここに `demo-trash` というダミーフォルダを用意しました。このフォルダを丸ごと削除するような操作を Claude Code に提案させます。

（「demo-trashフォルダを削除してください」と依頼する）

Claude Code が `rm -rf demo-trash` というコマンドを提案してきました。

見てください。このコマンドは、フォルダとその中身をすべて完全に削除します。Macのゴミ箱には入りません。実行したら元には戻せません。

だからこそ、デフォルトモードでは必ずここで確認が入ります。内容を読まずにEnterを押さないでください。

（拒否して操作を止める）

今回はキャンセルしました。このように、一度立ち止まって内容を読む習慣が、事故を防ぎます。

## [P4] クロージング — 安全に使う習慣を身につけよう

まとめると、3つの習慣だけ覚えてください。

「**モードは場面で選ぶ**」「**承認画面は必ず読む**」「**迷ったらCtrl+Eで意図を確認する**」。

フルオートモードは怖がる必要はありません。ただし、大切なファイルが入っていないサンドボックス環境や、テスト用のフォルダに限定して使うのが安全です。

なお、環境変数やAPIキーの扱いは、S2以降でSupabaseに接続するときに実際の場面で学びます。今は「そういうものがある」とだけ覚えておいてください。
