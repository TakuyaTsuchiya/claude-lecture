修了課題: 自分のために何か1つ作ってみよう

ここまでで、Claude Codeでできることを広く見てきました。仕上げに、自分のために何か1つ作ってみましょう。

あなた自身が「これ、面倒だな」「こうだったらいいのに」と感じている問題への解決であればテーマは自由です。生活のため、子どものため、仕事のため、用途は問いません。便利ツール・アプリ・HTMLページ、形式も問いません。

条件

項目 内容
問題意識 業務 or 生活で、自分が実際に感じている課題から出発すること
問題の大きさ 問いません（「ちょっと面倒」レベルで十分です）
形式 便利ツール・アプリ・HTMLページ、何でも可
NG① 講座で扱った例のコピペ（自分の文脈に置き換えてください）
NG② 機密情報・個人情報の混入（業務データは仮データに、生活のものでも氏名・住所等は伏せてください）

提出するもの

以下をまとめてLMSに提出してください。

  1. 動作スクリーンショット: 動作している様子のスクショを好きなだけ添付してOK
  2. 作ったもの: なにが問題で、どう解決したか
  3. input と output（あれば）
  4. 工夫した点

ルーブリック（自己チェック用）

採点ではなく、「良い提出」の目安として使ってください。修了は提出確認のみで認定します。

観点 修了ライン（必須） 良い提出 事例紹介候補
問題意識 自分の問題から出発している 問題の背景・なぜ困っていたかが具体的 他の受講生にも共感される普遍的な問題
解決の着地 動くものができている 自分の問題を実際に解決できている 提出後も自分で使い続けたいと思える
Claude Codeの活用 Claude Codeを使って作っている プロンプトの工夫・反復の跡が見える 動画では扱っていない使い方を発見している
工夫 工夫した点を1つ以上書いている 工夫の中身（プロンプト・設計・反復）が具体的 他の受講生のヒントになる学びが言語化されている

運用

  • 提出確認のみで修了: 合否判定はありません
  • 事後、事例として紹介したい場合のみ個別連絡: その際にコードや詳細を見せていただきたい場合は、改めてご相談します
  • 提出を確認したら修了処理します。お疲れさまでした!
