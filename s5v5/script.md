# S5V5 台本 — 経費管理アプリのフロントエンドをスペック駆動で作ろう

- **想定尺**: 約6分（うち実演 約4分）
- **実演**: あり（プロンプト①で設計→プロンプト②で実装→GitHub push→Vercelデプロイ）

---

## [P1] カバー — 経費管理アプリのフロントエンドを作ろう

この動画では、経費管理アプリをスペック駆動で開発します。Claude Codeでアプリ開発用のCLAUDE.mdと、仕様書であるspec.mdを作り、それを元に実装まで行います。

---

## [P2] TODAY'S GOAL — アプリのスペック駆動開発を体験する

流れは2ステップです。まず**設計**として、Claude CodeでCLAUDE.mdとspec.mdを生成します。次に**開発**として、その2つを元に経費管理アプリを生成します。

スペック駆動とは、先に仕様書を書いてから、それをもとに実装に入る手法です。いきなりプロンプトから書き始めるのではなく、仕様書で何を作るのか、を明確にするのがポイントです。

---

## [P3] WHY — SaaS機能の核、データのCRUDを学ぶ

経費管理のSaaSに月額で払っている会社は多いです。

経費管理SaaSを手元で動くアプリを作ることで、どのような仕組みなのかを学びます。アプリの基本であるCRUD、すなわちデータの登録、読み取り、更新、削除の機能を開発することがゴールです。

---

## [P4] CLAUDE.md — 基本の7項目

CLAUDE.mdに書くことは、基本の7項目で整理できます。技術スタック、ファイル構成、コード規約、データの形、デザインルール、禁止事項、作業フロー。この7つを基本としつつ、作るアプリの用途に合わせて改変します。

spec.mdにはアプリの画面構成、CRUD操作フロー、データ項目、データ永続化の方針を書きます。この2つが揃うと、Claude Codeは迷わず実装に入れます。

---

## [P5] PROMPT ① — CLAUDE.md/spec.md 生成プロンプト（一部）

（画面を Claude Code デスクトップ版に切り替え、新しいプロジェクト用ディレクトリを作成）

最初に、CLAUDE.mdとspec.mdを作るためのプロンプトを渡します。プロンプトは長いので概要欄に貼っておきます。

（画面に詳細版プロンプトを貼り付けて見せる）

> 経費管理アプリを作ります。
> いきなり実装せず、
> CLAUDE.md と spec.md を先に書いてください。
>
> 画面：左に一覧、右に詳細のツーカラム
> 保存：ブラウザの localStorage
> 技術：HTML + JS + Tailwind CDN のみ
>
> ……（以下、データ項目・CRUD操作フロー・デザイン方針・対象外の機能）

（Claude Codeにプロンプトを投入。CLAUDE.mdとspec.mdが生成されていく）

生成された2つのファイルを確認します。

（CLAUDE.mdを画面に表示、7項目を上から眺める）

技術スタックはHTMLとJavaScript、Tailwind CDN。ディレクトリはindex.html / app.js / styles.cssの3枚構成。規約や禁止事項も埋まっています。

（spec.mdを画面に表示、画面構成とCRUD操作フローを眺める）

spec.mdも画面構成とCRUD操作が書かれていて、このまま実装に渡せます。

---

## [P6] PROMPT ② — アプリをビルドするプロンプト

続いて、先ほど作った2枚のドキュメントを元に、実装を依頼します。

> CLAUDE.md と spec.md を読んで、
> 経費管理アプリを実装してください。
>
> index.html / app.js / styles.css の
> 3ファイルで完結させてください。
>
> データはブラウザのlocalStorageに保存します。
> 実装が終わったらGitHubにプッシュしてください。

アプリの詳細は全部CLAUDE.mdとspec.mdに書いてあるので、ここでは「実装して」と頼むだけで済みます。

（Claude Codeにプロンプトを投入。実装とGitHubへのpushまで一気に進む）

pushが終わったら、S5V2でやったのと同じ手順で、このリポジトリをVercelに連携します。

（Vercelダッシュボード → Add New → Project → 今作ったリポジトリを選択 → Import → Deploy）

ビルドが終わるとURLが発行されます。一度Importしてしまえば、次からはpushだけで自動デプロイされます。

（発行されたURLを開き、経費の登録、一覧確認、編集、削除を順に操作）

左に経費の一覧、右に詳細。データはブラウザのlocalStorageに入っているので、リロードしても消えません。

---

## [P7] TEMPLATE — CLAUDE.mdの基本の形

最後に、CLAUDE.mdの基本の形をお見せします。スライドに出している7項目をそのまま見出しにした、シンプルなMarkdownです。この型を覚えておくと、次にアプリを作るときも同じ枠組みで書き始められます。

今回作った経費管理アプリはフロントエンドだけで完結しているので、データはブラウザの中にしか残らず、他の人と共有できません。S6でSupabaseのデータベースと繋ぐと、どの端末からでも同じデータが見えるようになります。

これで、スペック駆動でCRUDアプリを作ることができました。
