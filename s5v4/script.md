# S5V4 台本 — 名刺管理アプリのフロントエンドをスペック駆動で作ろう

- **想定尺**: 約5分（うち実演 約3.5分）
- **実演**: あり

---

## [P1] カバー — 名刺管理アプリのフロントエンドを作ろう

この動画では、名刺管理アプリをスペック駆動で開発します。Claude Codeでアプリ開発用のCLAUDE.mdと、仕様書であるspec.mdを作り、それを元に実装まで行います。Claude Codeでアプリの基本機能であるCRUD、すなわちCreate, Read, Update, Deleteを開発することがゴールです。

---

## [P2] TODAY'S GOAL — スペック駆動でアプリ開発を体験する

流れは3ステップです。アプリ開発用のCLAUDE.mdを書き、spec.mdを書いて画面と機能を決め、Claude Codeと協働でアプリを組み上げます。

アプリ開発にはスペック駆動という考え方があります。先に仕様書を書き切ってから実装に入る進め方です。Claude Codeと仕事をするときは、いきなりプロンプトから書き始めるのではなく、CLAUDE.mdとspec.mdで土台を用意するのが鍵です。Claude Codeは渡されたドキュメント通りに動くので、ドキュメントの質がそのまま出力の質になります。

---

## [P3] WHY — 月額数千円のSaaSが自前で作れる時代

名刺管理SaaSの機能の核は、データを登録・一覧（検索含む）・編集・削除する4つの操作、CRUDです。手元で動くアプリを作ることで、こうしたSaaSがどのような仕組みなのかを学びます。この機能の核だけなら、スペック駆動とClaude Codeの組み合わせで動くものを作ることができます。

---

## [P4] CLAUDE.md — アプリ開発用CLAUDE.mdに書くこと

（画面を Claude Code デスクトップ版に切り替え、新しいプロジェクト用ディレクトリを作成）

アプリ開発用のCLAUDE.mdに記載するのは技術スタック、ディレクトリ構成、コーディング規約、データ構造、デザイン規約、やってはいけないこと、の6項目。わからないことが多いと思いますが、とりあえず進めて動くものを作り、あとから振り返ってください。概要欄にCLAUDE.mdを貼っておきます。

（CLAUDE.mdファイルを画面に表示）

続いてClaude Codeとspec.mdを書きます。Claude Codeが迷わず実装できるよう、左に名刺の一覧、右に詳細とフォームの2カラム画面構成、CRUDの各操作フロー、localStorageに保存する方針を言語化し、仕様書を作成します。

---

## [P5] PROMPT

（スライドをP5に切り替え、プロンプトを読み上げ）

> CLAUDE.md と spec.md を読んで、
> 名刺管理アプリを実装してください。
>
> index.html / app.js / styles.css の
> 3ファイルで完結させてください。
>
> データはブラウザのlocalStorageに保存します。
> 実装が終わったらGitHubにプッシュしてください。

次に、実装を開始するプロンプトをClaude Codeに渡します。CLAUDE.mdとspec.mdに詳細が明記されているので、ここでは「実装して」と頼むだけで済みます。

（Claude Codeにプロンプトを投入。ファイルが生成されていく）

実装が終わったら、ブラウザで開いて動かしてみます。

（実装したアプリを開き、カードを選択、新規登録、編集、削除を順に操作）

左に名刺の一覧、右に詳細。CRUDが全部動いています。データはブラウザのlocalStorageに入っているので、リロードしても消えません。

続いて、GitHubへのプッシュもClaude Codeに頼みます。

> 実装が終わりましたので、GitHubにpushしてください。

（Claude Codeがpushを実行）

GitHubへプッシュすると、Vercelで自動デプロイが走り、URLが発行されます。

（発行されたURLを開く）
